Introduction to Java Programming


Published on Feb 11, 2024
Rajesh Bhola

Introduction

Welcome to bhola{Code}Camp, In this article we will learn about what is java, history of java, features of java, uses of java, what is the differences between c++ and java and classifications of java. So without any delay let's get started...


What is Java

  • Java is a popular high-level, object-oriented programming language, which was originally developed by Sun Microsystems.

  • It was designed to be platform-independent, meaning that Java programs can run on any device or operating system that has a Java Virtual Machine (JVM) installed. ("Write Once, Run Anywhere" (WORA)).


History of Java

  • The story of Java begins back in the early 1990s when a group of developers at Sun Microsystems, led by James Gosling, began creating a new programming language and platform.

  • In 1991, James Gosling, Mike Sheridan, and Patrick Naughton began working on a project initially called "Oak" at Sun Microsystems.

  • The project aimed to create a programming language for consumer electronics.

  • The name "Java" was later chosen due to trademark issues with the name "Oak."

  • On May 23, 1995, Sun Microsystems announced the Java platform publicly.

  • Java 1.0 was released in January 1996, marking the first official version of the Java platform.

  • In 2010, Oracle Corporation acquired Sun Microsystems, including the Java platform and related technologies.

  • Java 8, released in 2014, introduced significant features such as lambda expressions, the Stream API, and the java.time package.

  • Oracle open-sourced the Java platform by contributing the code to the OpenJDK project, an open-source implementation of the Java platform.


Features of Java


  • Simple

    • Java syntax is straightforward and easy to understand.

    • Java does not have pointers. By eliminating pointers, Java simplifies memory management and reduces the risk of common programming mistakes.

    • Java manages memory automatically through garbage collection.

    • Java provides strong type checking at compile time and runtime.

    • Java follows the object-oriented programming paradigm.

    • Java comes with a vast standard library (Java API) that provides ready-to-use classes and methods for common tasks.

    • Java programs are compiled into bytecode, which can be executed on any device or operating system with a Java Virtual Machine (JVM) installed.


  • Object-Oriented

    • Java is considered object-oriented primarily because it follows the core principles of object-oriented programming (OOP).

    • Java supports Encapsulation, Inheritance, Polymorphism, Abstraction, Objects and Classes.


  • Platform Independent

    • Java compiles source code into bytecode that is independent of the underlying hardware and operating system.

    • Bytecode is executed by the Java Virtual Machine (JVM), which is a software-based interpreter or runtime environment that translates bytecode into machine-specific instructions at runtime.

    • The combination of bytecode and the JVM enables Java programs to be "write once, run anywhere" (WORA).


  • Secured

    • The sandbox restricts the actions that Java code can perform, such as accessing the file system, network, or system resources, thereby preventing malicious behavior.

    • Before executing bytecode, the Java Virtual Machine (JVM) performs bytecode verification to ensure that it adheres to certain security rules.

    • Java applications can optionally use a Security Manager to enforce fine-grained security policies.

    • Oracle, regularly releases security updates and patches to address known vulnerabilities and security issues in the Java platform.


  • Portable

    • Java is considered portable because of its platform independence and the "Write Once, Run Anywhere" (WORA) principle.


  • Robust

    • Java provides strong type checking at both compile time and runtime.

    • Java manages memory automatically through garbage collection.

    • There is a lack of pointers that avoids security problems Java's exception handling mechanism allows developers to handle errors and unexpected conditions gracefully.


  • Architectural Neutral

    • Java's architecture neutrality is maintained by ensuring that certain features, such as the size of primitive data types, remain consistent across different hardware architectures and platforms.

    • This consistency helps ensure that Java programs behave predictably and consistently regardless of the underlying hardware or operating system.

    • Java maintains its architecture neutrality, allowing developers to write code once and run it anywhere without having to worry about platform-specific details or optimizations.


  • High Performance

    • Java employs a Just-In-Time (JIT) compiler that converts bytecode into native machine code at runtime.

    • The HotSpot JVM, developed by Oracle, is a high-performance Java Virtual Machine implementation that includes advanced runtime optimizations such as method inlining, loop unrolling, and adaptive optimization.

    • Java's garbage collection mechanism automatically manages memory allocation and deallocation, reducing the risk of memory leaks and memory-related performance issues.

    • Modern garbage collectors, such as the G1 Garbage Collector introduced in Java 7 and the Z Garbage Collector introduced in Java 11, are optimized for low-latency and high-throughput performance.

    • Multithreading enables Java applications to take advantage of modern multi-core processors and improve performance through parallelism and concurrency.


  • Distributed

    • Java provides extensive networking support through its standard library (Java API), which includes classes and interfaces for building networked applications.

    • Java's platform independence enables Java applications to run on any device or operating system with a Java Virtual Machine (JVM) installed.

    • Java includes Remote Method Invocation (RMI), a mechanism for invoking methods on remote objects over a network.

    • RMI uses the Java Remote Method Protocol (JRMP) for communication between Java objects in distributed systems.

    • Java supports various message-oriented middleware (MOM) technologies, such as Java Message Service (JMS), for building asynchronous, reliable, and scalable messaging systems.

    • Java ecosystem offers various distributed computing frameworks and libraries, such as Apache Hadoop, Apache Spark, and Apache Kafka, for building distributed data processing, analytics, and messaging systems.


  • Multi-threaded

    • Multithreading enables Java applications to maintain responsive user interfaces while performing time-consuming tasks in the background. By running tasks on separate threads, the user interface remains interactive and does not freeze or become unresponsive.

    • Multithreading allows Java applications to take advantage of modern multi-core processors by executing multiple threads simultaneously. This maximizes CPU utilization and improves overall system performance, especially for CPU-bound tasks Multithreading enables asynchronous programming in Java, where tasks can execute independently and asynchronously without blocking the main thread.

    • Multithreading supports scalable designs in Java applications, allowing them to scale up to handle increasing workloads and user concurrency.

    • Multithreading enables efficient resource sharing and management in Java programs. Threads can share data and resources within the same process, facilitating communication and coordination between different parts of the application.


  • Dynamic

    • Java supports dynamic class loading, which allows classes to be loaded into the JVM at runtime. This feature enables applications to dynamically load and instantiate classes based on runtime conditions or user input. It is commonly used in frameworks like Java Reflection and Class.forName().

    • Java provides support for dynamic proxies, which allow the creation of proxy objects that implement one or more interfaces at runtime.


  • Interpreted

    • Java source code is first compiled by the Java compiler (javac) into an bytecode. This bytecode is platform-independent and can be executed on any system with a Java Virtual Machine (JVM) installed.

    • The Java bytecode is then interpreted by the JVM at runtime. The JVM reads and executes the bytecode instructions, translating them into native machine code instructions that are specific to the underlying hardware architecture and operating system.


Uses of Java

  • Java is used to develop desktop applications, using frameworks, such as Swing and JavaFX.

  • Java is widely used for building server-side web applications and web services, such as servlets, JavaServer Pages (JSP), JavaServer Faces (JSF), and Enterprise JavaBeans (EJB). Popular web frameworks like Spring and Jakarta EE (formerly Java EE) simplify web development in Java.

  • Java is used for developing Android applications.

  • Java is widely used for developing large-scale enterprise applications, including customer relationship management (CRM) systems, enterprise resource planning (ERP) systems, and business process management (BPM) solutions.

  • Java is used for developing web services and APIs, such as Java API for RESTful Web Services (JAX-RS) and Spring Boot.

  • Java is used in the field of big data and analytics for developing data processing, analytics, and machine learning applications.

  • Java is used for developing software for embedded systems and Internet of Things (IoT) devices.

  • Java is used in scientific and research computing for developing computational and data analysis tools, simulations, and modeling applications.

  • Java is used for developing desktop and mobile games using libraries and frameworks such as LibGDX and LWJGL (Lightweight Java Game Library).


Classifications of Java


Java 2 Standard Edition (J2SE)

Java 2 Standard Edition (J2SE) is a platform for building and deploying desktop applications and applets. This platform used for developing general-purpose desktop, server-side, and command-line applications.


Java 2 Enterprise Edition (J2EE)

Java 2 Enterprise Edition (J2EE) is a platform for building and deploying enterprise-level applications, particularly web-based and distributed enterprise applications.
Technologies such as Servlets, JavaServer Pages (JSP), Enterprise JavaBeans (EJB), Java Persistence API (JPA), Java Message Service (JMS), Java Transaction API (JTA), and others.
Java EE servers, such as Apache Tomcat, WildFly, and IBM WebSphere, provide environments for deploying and running Java EE applications.


Java 2 Micro Edition (J2ME)

Java 2 Micro Edition (J2ME) is a platform developed by Sun Microsystems (now Oracle Corporation) for building applications for small, resource-constrained devices such as mobile phones, PDAs, set-top boxes, smart cards, and embedded systems.



Differences between C++ and Java


C ++

  • C++ has a more complex syntax compared to Java, support for features like manual memory management (via pointers), multiple inheritance, and operator overloading.

  • C++ requires explicit memory management using pointers C++ code is typically compiled directly into machine code specific to the target platform's architecture and operating system. This makes C++ applications platform-dependent.

  • C++ applications are typically compiled directly into executable files that are run by the operating system. They do not require a separate runtime environment.

  • C++ is commonly used for systems programming, game development, performance-critical applications, and applications that require low-level access to hardware resources.


Java

  • Java has a simpler syntax and enforces object-oriented programming principles more rigorously does not support manual memory management (garbage collection is automatic), multiple inheritance, or operator overloading.

  • Java uses automatic memory management through garbage collection.

  • Java code is compiled into platform-independent bytecode, which is executed by the Java Virtual Machine (JVM). This allows Java applications to run on any platform with a compatible JVM, making them platform-independent.

  • Java applications require the Java Runtime Environment (JRE) or Java Development Kit (JDK) installed on the target system to run. The JRE includes the JVM, class libraries, and other components necessary for executing Java applications.

  • Java is widely used for enterprise application development, web development, mobile app development (Android), server-side programming, and applications where platform independence and security are important.
