  • Maximum array size is value of int size 
    • 
    int[] a22 = new int[2147483647]; // may give OutOfMemoryError

    OutOfMemoryError: occurs because the JVM (Java Virtual Machine) cannot allocate an array of size 2147483647 due to memory limitations. This error happens because the maximum size of an array in Java is Integer.MAX_VALUE, which is 2147483647.

    When you create an array in Java, the JVM needs to allocate memory for the array elements. Each element of an integer array (int[]) takes up 4 bytes (32 bits) of memory. So, for an array of size 2147483647, the JVM would need to allocate approximately 2147483647 * 4 = 8589934588 bytes, which is roughly 8 GB of memory.

    The JVM has limits on the amount of memory it can allocate for an array, and these limits are often lower than the total available memory on the system. When you attempt to create an array that exceeds these limits, the JVM throws an OutOfMemoryError to indicate that it cannot allocate the requested memory.
